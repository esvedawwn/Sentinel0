PRODUCT PURPOSE

Sentinel Organizer Lite is a privacy-conscious native macOS file intelligence application.

It scans files, classifies them, identifies clutter and risk, and presents recommendations.

AI should improve:

- file classification

- document summaries

- semantic tags

- suggested destinations

- cleanup explanations

- natural-language search

- project grouping

- duplicate review

- organisation recommendations

AI must never directly delete, move, rename or modify files.

AI may only:

- classify

- summarise

- tag

- explain

- score confidence

- suggest actions

All file operations must remain controlled by the existing SafetyEngine and require explicit user confirmation.

SAFETY REQUIREMENTS

These are mandatory.

1. AI never executes filesystem actions directly.

2. AI cannot bypass SafetyEngine.

3. AI recommendations must be reviewable.

4. Every recommendation must explain why.

5. Confidence must be displayed.

6. Low-confidence results must be labelled clearly.

7. Sensitive files must default to local-only processing.

8. API keys must not be committed.

9. File content must not be uploaded automatically.

10. The app must remain functional when AI is disabled.

11. Any destructive action must be explicit, reversible where possible, and confirmed.

12. Add a visible privacy explanation in Settings.

────────────────────────────────────────

ARCHITECTURE REQUIREMENTS

- Keep AI logic out of SwiftUI views.

- Use protocols for providers.

- Use dependency injection.

- Keep models Codable where practical.

- Make async operations cancellable.

- Use async/await.

- Update UI state on the main actor.

- Add error types with useful user-facing messages.

- Avoid singletons unless clearly justified.

- Do not duplicate existing classifier logic.

- Integrate with FileClassifier rather than replacing it recklessly.

- Preserve SafetyEngine as the authority for file actions.
